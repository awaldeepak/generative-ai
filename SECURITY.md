## Security Policy

If you discover a security vulnerability in this repository, please follow these steps to report it responsibly:

1. Do not create a public issue for security-sensitive information.
2. Prefer contacting the maintainer directly or open a private issue on GitHub (if available) and mark it as `security`.
3. Provide a clear description, reproduction steps, and any PoC code or logs.
4. If the issue includes leaked credentials or keys, revoke/rotate them immediately.

Maintainer contact: open an issue with the `security` label or email the project owner.

Disclosure timeline: We aim to acknowledge reports within 3 business days and to provide a remediation plan as soon as possible.
