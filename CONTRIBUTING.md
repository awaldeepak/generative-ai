## Contributing

Thanks for your interest in contributing. A few guidelines to keep the project healthy:

- Keep changes small and focused; open issues for larger proposals.
- Follow existing code style and add comments where helpful.
- Add tests or a small reproducible example for behavior changes.
- Do not commit secrets or credentials. Use environment variables and add local `.env` to `.gitignore`.
- All contributions are subject to the repository `LICENSE`.

Workflow

1. Fork the repository.
2. Create a feature branch: `git checkout -b my-feature`.
3. Commit changes with clear messages.
4. Open a pull request describing the change and link relevant issues.
